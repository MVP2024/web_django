# Название Проекта: Django Skystore

## Описание Проекта

Этот проект представляет собой простое веб-приложение, разработанное на фреймворке Django. Целью проекта является создание базовой структуры для онлайн-магазина или каталога товаров ("Skystore"), включая главную страницу и страницу контактов с формой обратной связи.

Проект использует:
- Django для бэкенда и шаблонизации.
- Bootstrap для базового оформления и адаптивности (подключен через CDN).
- Poetry для управления зависимостями и виртуальным окружением.
- PostgreSQL в качестве базы данных (настраивается через `.env`).
- Redis для кэширования.
- Фикстуры для загрузки начальных данных.

## Структура Проекта

Основные файлы и директории проекта:

-   `.env` - Файл с переменными окружения для настроек проекта, включая параметры базы данных, email, Redis и флаг кэширования (игнорируется Git).
-   `.git/` - Файлы системы контроля версий Git.
-   `.gitignore` - Правила игнорирования файлов для Git.
-   `.idea/` - Файлы настроек IDE PyCharm.
-   `.venv/` - Виртуальное окружение, созданное Poetry.
-   `blog/` - Django приложение "blog" (для статей или новостей).
-   `catalog/` - Django приложение "catalog" (основной функционал магазина).
    -   `fixtures/` - Директория для файлов фикстур.
        -   `initial_data.json` - Файл с начальными данными для моделей Category и Product.
    -   `management/commands/` - Директория для пользовательских команд управления.
        -   `del_all_and_load_new_data.py` - Команда для очистки данных и загрузки фикстуры.
    -   `migrations/` - Файлы миграций базы данных.
    -   `templates/` - HTML шаблоны приложения.
    -   `views.py` - Контроллеры (представления) приложения.
    -   `urls.py` - URL-маршруты приложения.
    -   `models.py` - Модели данных (Category, Product, ContactInfo).
    -   `admin.py` - Конфигурация админ-панели.
    -   `apps.py` - Конфигурация приложения.
    -   `forms.py` - Формы для создания/редактирования продуктов и контактов.
    -   `services.py` - Сервисные функции, включая логику кэширования.
    -   `templatetags/` - Пользовательские теги шаблонов.
    -   `tests.py` - Тесты приложения.
-   `config/` - Основная конфигурация проекта.
    -   `settings.py` - Основные настройки проекта, включая загрузку `.env`, настройки кэширования.
    -   `urls.py` - Главные URL-маршруты проекта.
    -   `wsgi.py` - WSGI конфигурация для развертывания.
    -   `asgi.py` - ASGI конфигурация для асинхронных приложений.
-   `manage.py` - Утилита командной строки Django.
-   `media/` - Директория для пользовательских медиа-файлов (изображения продуктов, аватары пользователей). Игнорируется Git.
-   `poetry.lock` - Зафиксированные версии зависимостей проекта.
-   `pyproject.toml` - Конфигурация Poetry и список зависимостей.
-   `README.md` - Описание проекта и инструкции.
-   `screenshots/` - Директория для скриншотов.
-   `server.py` - Возможно, файл для запуска сервера (не стандартный для Django).
-   `static/` - Директория для статических файлов приложения (CSS, JS, изображения).
-   `staticfiles/` - Директория для собранных статических файлов (игнорируется Git).
-   `users/` - Django приложение "users" (для управления пользователями).
    -   `forms.py` - Формы для регистрации, входа и редактирования профиля.
    -   `models.py` - Кастомная модель пользователя.
    -   `templates/` - HTML шаблоны для страниц пользователя.
    -   `views.py` - Представления для аутентификации и управления профилем.

## Установка и Запуск

Для установки и запуска проекта выполните следующие шаги:

1.  **Клонирование репозитория (если применимо):**
    Если проект находится в удаленном репозитории, клонируйте его:
    ```
    git clone <URL_вашего_репозитория>
    cd web_django-develop # Или имя вашей корневой директории
    ```
    Если у вас уже есть файлы локально, просто перейдите в корневую директорию проекта:
    ```
    cd D:/Project/web_django-develop
    ```

2.  **Настройка переменных окружения:**
    Создайте файл `.env` в корневой директории проекта, скопировав содержимое из `.env.sample`. Заполните необходимые данные для подключения к вашей базе данных PostgreSQL, настройки Email, Redis и другие параметры.
    Пример `.env`:
    ```dotenv
    SECRET_KEY='ваш_секретный_ключ'
    DEBUG=True
    DATABASE_NAME='ваша_бд'
    DATABASE_USER='ваш_пользователь'
    DATABASE_PASSWORD='ваш_пароль'
    DATABASE_HOST='ваш_хост'
    DATABASE_PORT='5432'

    EMAIL_BACKEND='django.core.mail.backends.console.EmailBackend'
    EMAIL_HOST='smtp.example.com'
    EMAIL_PORT='587'
    EMAIL_USE_TLS='True'
    EMAIL_USE_SSL='False'
    EMAIL_HOST_USER='your_email@example.com'
    EMAIL_HOST_PASSWORD='your_email_password'

    REDIS_URL='redis://127.0.0.1:6379/1'
    REDIS_URL_PRODUCTS='redis://127.0.0.1:6379/2'
    CACHE_ENABLED='True' # Установите 'False' для отключения кэширования
    ```

3.  **Установка Poetry:**
    Если у вас не установлен Poetry, установите его согласно официальной документации: [https://python-poetry.org/docs/#installation](https://python-poetry.org/docs/#installation)

4.  **Установка зависимостей:**
    Poetry прочитает файл `pyproject.toml` и установит все необходимые зависимости в виртуальное окружение.
    ```
    poetry install
    ```
    Эта команда также создаст и активирует виртуальное окружение `.venv` в корне проекта, если оно еще не существует.

5.  **Активация виртуального окружения:**
    Активируйте виртуальное окружение Poetry:
    ```
    poetry shell
    ```
    Теперь вы находитесь внутри виртуального окружения, и все команды Python будут использовать установленные зависимости.

6.  **Применение миграций базы данных:**
    Примените миграции для создания необходимых таблиц в вашей базе данных PostgreSQL:
    ```
    python manage.py migrate
    ```

7.  **Загрузка начальных данных (опционально):**
    Вы можете загрузить тестовые данные из фикстуры, используя пользовательскую команду:
    ```
    python manage.py del_all_and_load_new_data
    ```
    Эта команда очистит существующие данные в моделях Category и Product и загрузит данные из файла `catalog/fixtures/initial_data.json`.

8.  **Запуск сервера разработки:**
    Запустите локальный сервер разработки Django:
    ```
    python manage.py runserver
    ```

    Сервер будет доступен по адресу `http://127.0.0.1:8000/`.

## Использование

После запуска сервера вы можете перейти по следующим адресам в браузере:

*   **Главная страница (Каталог):** `http://127.0.0.1:8000/` - Отображает последние 5 добавленных продуктов.
*   **Страница контактов:** `http://127.0.0.1:8000/contacts/` - Содержит форму обратной связи и контактную информацию (загружается из фикстуры).
*   **Список категорий:** `http://127.0.0.1:8000/categories/`
*   **Список продуктов по категории (без кэша):** `http://127.0.0.1:8000/categories/<pk_категории>/products/`
*   **Список продуктов по категории (с кэшем):** `http://127.0.0.1:8000/categories/<pk_категории>/products/cached/`
*   **Детали продукта:** `http://127.0.0.1:8000/products/<pk_продукта>/` - Страница кэшируется, если `CACHE_ENABLED='True'`.
*   **Добавить продукт:** `http://127.0.0.1:8000/products/create/` (требуется авторизация).
*   **Редактировать продукт:** `http://127.0.0.1:8000/products/<pk_продукта>/update/` (требуется авторизация и быть владельцем продукта).
*   **Удалить продукт:** `http://127.0.0.1:8000/products/<pk_продукта>/delete/` (требуется авторизация и быть владельцем продукта или иметь разрешение `catalog.delete_product`).
*   **Отменить публикацию продукта:** `http://127.0.0.1:8000/products/<pk_продукта>/unpublish/` (требуется авторизация и иметь разрешение `catalog.can_unpublish_product`).
*   **Регистрация:** `http://127.0.0.1:8000/users/register/`
*   **Вход:** `http://127.0.0.1:8000/users/login/`
*   **Выход:** `http://127.0.0.1:8000/users/logout/`
*   **Редактирование профиля:** `http://127.0.0.1:8000/users/profile/edit/` (требуется авторизация).
*   **Админ-панель:** `http://127.0.0.1:8000/admin/` - Здесь вы можете управлять всеми моделями.
На странице контактов вы найдете форму обратной связи. Заполнение и отправка этой формы приведет к выводу введенных данных в консоль, где запущен сервер Django.

## Дополнение по ДЗ

На данный момент проект значительно расширен и включает следующие ключевые функции и улучшения:

### Основной функционал (Каталог)
- **Модели данных:** `Category`, `Product`, `ContactInfo` с расширенными полями.
- **CRUD операции для продуктов:** Реализованы представления для создания (`ProductCreateView`), просмотра деталей (`ProductDetailView`), редактирования (`ProductUpdateView`) и удаления (`ProductDeleteView`) продуктов.
- **Отображение продуктов:**
    - Главная страница отображает последние 5 опубликованных продуктов.
    - Страницы со списком продуктов по категориям (`ProductListByCategoryView`).
    - Отдельное представление (`CachedProductListByCategoryView`) для демонстрации кэшированного списка продуктов по категории.
- **Контакты:** Страница контактов с формой обратной связи и выводом данных в консоль.
- **Управление публикацией:** Функция `unpublish_product` для отмены публикации продукта, доступная по специальному разрешению.

### Управление пользователями
- **Кастомная модель пользователя:** Используется `users.User` с `email` в качестве `USERNAME_FIELD`.
- **Аутентификация:** Реализованы регистрация (`UserRegisterView`), вход (`UserLoginView`) и выход (`UserLogoutView`) пользователей.
- **Профиль пользователя:** Возможность редактирования профиля (`UserProfileEditView`), включая аватар, телефон и страну.
- **Приветственное письмо:** Отправка приветственного письма на email после успешной регистрации.

### Кэширование (Redis)
- **Интеграция Redis:** Настроен Redis в качестве бэкенда для кэширования.
- **Глобальный переключатель кэширования:** Переменная окружения `CACHE_ENABLED` в `.env` и `settings.py` позволяет включать/отключать кэширование.
- **Кэширование страниц:** Страница детализации продукта (`ProductDetailView`) кэшируется на 5 минут, если `CACHE_ENABLED` активно.
- **Низкоуровневое кэширование:** Сервисная функция `get_products_by_category_cached` использует низкоуровневое кэширование для списка продуктов по категории.

### Админ-панель
- **Отображение изображений:** В админ-панели для `Category` и `Product` отображаются миниатюры изображений.
- **Пользовательские действия:** Добавлено действие "Отменить публикацию выбранных продуктов" (`unpublish_selected_products`) для `ProductAdmin`.
- **Расширенная админка пользователей:** Настроена `UserAdmin` для управления кастомной моделью пользователя.

### Разрешения и Безопасность
- **Разрешения на уровне модели:** Добавлено кастомное разрешение `catalog.can_unpublish_product`.
- **Контроль доступа:** Использование `LoginRequiredMixin` и `UserPassesTestMixin` для ограничения доступа к CRUD операциям с продуктами (только владелец или пользователи с соответствующими разрешениями).

### Прочие улучшения
- **Пользовательские теги шаблонов:** Добавлен тег `has_perm` для проверки разрешений пользователя в шаблонах.
- **Управление зависимостями:** Проект использует Poetry.
- **База данных:** PostgreSQL, настроенная через переменные окружения.
- **Фикстуры:** Наличие `initial_data.json` и пользовательской команды `del_all_and_load_new_data` для загрузки начальных данных.
- **Статические и медиа-файлы:** Настройка для обслуживания в режиме разработки.
